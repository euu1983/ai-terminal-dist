AI Terminal — Privacy Policy

Last updated: 2026-04-28 · Applies to AI Terminal Android app (com.openclaw.aiterminal)

Who We Are

AI Terminal lets you remotely control terminal sessions on your computer (Claude Code / Cursor / Aider etc.) from your phone. This policy explains how we handle your data.

Data We Collect

DataPurposeStored
Device identifier (Android ID)Identify your phone for pairing persistence and 14-day free trial calculationRelay server
Public key (E2E encryption)Establish encrypted channel between your phone and computer; relay cannot decryptRelay server + paired computer daemon
Device nameDisplay in device management UI (e.g. "Xiaomi 13 Pro")Paired computer daemon + local device
FCM push tokenWake your computer via Google FCM when offlineRelay server
Subscription statusValidate Google Play subscriptionRelay server

Optional Telemetry & Feedback (Opt-in)

The following data is collected only if you enable it in Settings → Privacy. All three toggles are OFF by default; first-launch consent dialog respects your choice (declining keeps everything off).

DataPurposeProcessorRegion
Anonymous usage events (DAU, session length, feature use counters — no input content)Understand how features are used to prioritize improvementsGoogle Firebase AnalyticsGoogle global infrastructure
Crash reports (stack trace, device state, sanitized — sensitive paths/keys/IPs scrubbed before send)Identify and fix crashesGoogle Firebase CrashlyticsGoogle global infrastructure
User-submitted feedback (description, optional logs and screenshot, sanitized via 18+ regex client-side and again server-side)Process user-reported bugs and suggestionsOur own feedback endpointAsia + Americas (low-latency multi-region)

Sanitization: Before any logs or feedback text leaves your device, the client runs a 18-rule regex scrub that strips: API keys (Anthropic, OpenAI, Google, AWS, GitHub PATs), JWT tokens, file paths containing usernames, private network IPs, email addresses, phone numbers. The same scrub runs server-side as a second layer.

Anonymity: Telemetry uses a randomly generated install UUID (not Android ID, not your device identifier). We do not link telemetry to your subscription or pairing identity. You can wipe and regenerate the install UUID at any time via Settings → Privacy → Reset Telemetry ID.

Opt-out: Toggle off in Settings → Privacy at any time. Past data submitted while opt-in cannot be retroactively recalled (technical limitation), but no further data will be sent.

Data We Do NOT Collect

  • Terminal command content / AI conversation content — End-to-end encrypted (X25519 + AES-256-GCM). Only your phone and paired computer can decrypt. Neither the relay server nor we can see them.
  • Personal identity — We do not require account signup; we do not collect your name, email, phone number, or persistently log IP address (IP is only used for TCP connection setup).
  • Location — No GPS or network location collected.
  • Contacts / SMS / call logs — We don't request these permissions.
  • Advertising identifiers — No ad SDKs integrated.

Data Sharing

We do NOT share your data with advertisers, data brokers, or insurers.

Subprocessors (only if you opt in to telemetry/crash reports):

  • Google Firebase — Analytics and Crashlytics events. Subject to Google's Privacy Policy. You can opt out anytime in Settings → Privacy.
  • Our feedback endpoint — only the bug reports you explicitly submit. Stored on access-controlled servers, retained 90 days then auto-purged.

Exception: When legally required (e.g. court subpoena), we will disclose the minimum required by law.

Data Retention

  • Device ID + public key: Records on relay server are auto-cleaned within 90 days after you delete the app.
  • Local data (SharedPreferences / file cache): Deleted when you uninstall the app.
  • FCM token: Auto-invalidated by Google when you uninstall.

Children's Privacy

This app is not intended for children under 13. We do not knowingly collect data from children.

Your Rights

  • Access: View your device ID in the app's "Manage Devices" page.
  • Delete: Uninstalling the app clears local data; contact us (email) to remove records on the relay server.
  • Decline: You may deny any permission, but features (e.g. QR pairing, file transfer) will be unavailable.

Policy Updates

This policy may be updated. Significant changes will be notified in-app. Continued use after updates implies consent.

AI Terminal 隐私政策

最后更新: 2026-04-19 · 适用于 AI Terminal Android 应用 (com.openclaw.aiterminal)

我们是谁

AI Terminal 是一款让你从手机远程操控电脑终端 (Claude Code / Cursor / Aider 等 AI 工具) 的应用。本隐私政策说明我们如何处理你的数据。

我们收集的数据

数据类型用途存储位置
设备标识 (Android ID)识别你的手机用于跨连接保持配对状态、计算 14 天免费试用中继服务器 (relay)
公钥 (端到端加密)建立你手机和电脑之间的加密信道,中继服务器无法解密通信内容中继服务器 + 已配对的电脑端 daemon
设备名称在多设备管理界面显示 (例: "Xiaomi 13 Pro")已配对的电脑端 daemon + 你的手机本地
FCM 推送 token电脑离线时通过 Google FCM 推送通知唤醒中继服务器
订阅状态校验你的 Google Play 订阅有效性中继服务器

可选遥测 / 反馈数据 (默认关闭, 自愿启用)

以下数据只有在你于 设置 → 隐私 主动开启时才会收集。三个开关默认全部关闭; 首次启动同意框尊重你的选择 (拒绝即全部关闭, 不会偷偷开启)。

数据类型用途处理方区域
匿名使用事件 (DAU, 会话时长, 功能使用计数 — 不含任何输入内容)理解功能使用情况以决定改进优先级Google Firebase AnalyticsGoogle 全球基础设施
崩溃报告 (堆栈追踪, 设备状态; 敏感路径 / 密钥 / IP 已 sanitize)定位和修复崩溃问题Google Firebase CrashlyticsGoogle 全球基础设施
用户主动提交的反馈 (描述, 可选日志和截图; 客户端 18+ 条正则脱敏后再传输, 服务端二次脱敏)处理用户上报的 bug 和建议我们自有的 feedback 后端亚太 + 美洲 (多区域低延迟)

脱敏机制: 任何日志或反馈文本离开设备前, 客户端运行 18 条正则规则去除: API 密钥 (Anthropic / OpenAI / Google / AWS / GitHub PAT)、JWT token、含用户名的文件路径、内网 IP、邮箱、手机号。服务端再做一次同样的脱敏作为兜底。

匿名性: 遥测使用客户端生成的随机安装 UUID (不是 Android ID, 不是你的设备标识符)。我们不把遥测和你的订阅 / 配对身份做关联。你可以随时在 设置 → 隐私 → 重置遥测 ID 清除并重新生成。

退出: 在 设置 → 隐私 随时切换关闭。已提交的数据因技术限制无法追溯撤回, 但之后不再上报任何数据。

我们不收集的数据

  • 终端命令内容 / AI 对话内容 — 端到端加密 (X25519 + AES-256-GCM),只有你的手机和已配对电脑能解密。中继服务器和我们都看不到。
  • 个人身份信息 — 我们不要求注册账号,不收集你的姓名、邮箱、手机号、IP 地址 (IP 仅用于建立 TCP 连接,不持久化记录)。
  • 位置信息 — 我们不收集 GPS / 网络位置。
  • 通讯录 / 短信 / 通话记录 — 不申请相关权限。
  • 广告标识符 — 不集成任何广告 SDK。

权限使用说明

权限用途
相机 (CAMERA)扫描电脑屏幕上的二维码完成首次配对。我们不录制、不上传、不存储任何相机内容。
读取媒体 / 文件 (READ_MEDIA_IMAGES / READ_EXTERNAL_STORAGE)当你主动选择图片或文件上传到电脑时使用。我们只在你点击"上传"按钮时读取你选中的文件。
网络访问 (INTERNET / ACCESS_NETWORK_STATE)连接中继服务器和你的电脑端 daemon。

数据共享

我们不将你的数据共享给广告商、数据交易方、保险公司等任何第三方。

子处理者 (仅当你启用遥测 / 崩溃报告时):

  • Google Firebase — 处理 Analytics 与 Crashlytics 事件。受 Google 隐私政策 约束。可在 设置 → 隐私 随时退出。
  • 我们自有的 feedback 后端 — 仅你主动提交的 bug 反馈。存储在受访问控制保护的服务器上, 保留 90 天后自动清除。

例外情况: 法律强制要求时 (例如法院传票),我们会按法律最低限度披露。
